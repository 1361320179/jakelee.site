## 问题

很多个人站都是从博客开始，后来不断叠加零散页面，直到重构成本开始拖慢迭代速度。

## 做法

通过 route groups 把营销页面和 API 能力隔离开。内容先放在 Git 里管理，等需要时再让数据库表接管订阅和联系消息。

## 结果

最终得到一个可以继续演进到 Payload 或自定义后台的单一代码库，而不需要重写已有公开链接。
